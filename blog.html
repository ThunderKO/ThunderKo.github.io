教育管理系統(LMS)後端建構(一) – Moodle

Post By Admin
Category : Web,Server
comment15

前言

筆者也差不多從大學本科畢業了，讀書的時候總是從學校提供的網站去下載演講及導修筆記。對於畢業之後就沒有這樣的方便感到很可惜(其實是依賴了…呵呵)，所以就打算自己建構一個來自己一個用。不同的院校使用的教育管理系統(LMS)都不同，例如有：Moodle, BlackBoard, Canvas, 當然也有些學校的前端是學校自己建構的，這個我就不在這裏討論了(不過筆者非常歡迎讀者在下面留言還有什麼其他的LMS)。第一站就和大家討論一下Moodle的建構環節！

技術概覧

系統環境(OS Env.) ： Debian 9.5.0
網絡伺服器(Web Server) ： Apache
數據庫管理系統(DBMS) ： MySQL
伺服器開發語言 ： PHP
筆者嘅選擇考慮了穩定性而且捨難取易，為求一步到位。因此選擇了Debian。Debian對比起Ubuntu更加輕量，發佈的版本亦經過針對建構伺服器的測試而且更新速度比較慢。因此減省了不少的維護時間，也比較方便。在後端的技術棧(Technology Stack)上，使用了被普遍使用的”LAMP棧”(Linux, Apache, MySQL, PHP)。因為筆者打算第一篇的建構是為初學者而設。在往後我們討論BlackBoard, Canvas的建構，會和大家嘗試不同的技術棧！～

第一步：“爬棧” (安裝LAMP棧)

1. 下載MySQL和Apache配置軟件。

wget https://dev.mysql.com/get/mysql-apt-config_0.8.6-1_all.deb mysql-apt-config_0.8.6-1_all.deb
apt-get install gdebi-core
gdebi mysql-apt-config_0.8.6-1_all.deb
apt-get update>

2. 下載”LAMP棧”。

apt-get install apache2 php7.1 mysql-server php7.1-mysql libapache2-mod-php7.1 php7.1-gd php7.1-curl php7.1-xmlrpc php7.1-intl php7.1-zip php7.1-mbstring

3. 修正PHP配置檔案。

nano /etc/php/7.1/apache2/php.ini
【加以下的代碼】
extension=mysql.so
extension=gd.so
4. 檢查及除錯目前的進度。
/etc/init.d/apache2 restart
【打開瀏覽器在網址打localhost】

第二步：“攻棧” (建構MySQL數據庫管理系統)

1. 建立MySQL管理員(admin)。

mysqladmin -u root password “破解不了的密碼"

2. 登錄MySQL。
mysql -u root -p

3. 建立Moodle數據庫及更新。
mysql> CREATE DATABASE moodle CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
mysql> exit;
/etc/init.d/mysql restart

第三步：“登頂” (配置Moodle前端)

1.下載Moodle。(筆記是下載Moodle3.5.0的tar檔案)
2. 解壓縮Moodle。

cd /home/你的用戶名稱/Downloads
tar -zxvf moodle-lastest-35

3. 瀏覽去檔案的位置並建構Moodle檔案架構。

cd/var/www
mkdir moodledata
chmod -R 755 moodle
chmod -R 755 moodledata

第四步：“觀空” (配置Moodle到Apache網頁)

1. 修改Apache檔案。

cd/var/www
nano /etc/apache2/sites-available/default
【去第四行，改成】
DocumentRoot “/var/www/moodle"
【去第十行，改成】
Directory “/var/www/moodle"
【去第十七行，改成】
# RedirectMatch ^/$ /apache2-default/
/etc/init.d/apache2 restart

第五步：“望月” (從前端配置Moodle)

1. 瀏覽"http://locahost:moodle"。
2. 因為只是跟大家討論，所以筆者就不在這裏跟大家展現把配置好的系統掛在網域DNS裏。

第六步：“再遇圓月” (配置更新系統)

1. 因為Moodle需要一個時間性的功能(function)去進行維護或更新。筆者就打算把每次的維護時間設定為十分鐘。

crontab -u www-data -e
*/10 * * * * /usr/bin/php /var/www/moodle/admin/cli/cron.php >/dev/null

最後你就能把你的演講及導修筆記都放在Moodle了。
